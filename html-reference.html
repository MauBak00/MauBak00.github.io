Document and HTML


Description

Click on the title of each section to see the relevant W3Schools tutorial.

The Doctype element indicates to the browser that the document is written in Hypertext Markup Language.

The HTML element follows, and consists of the content of the website that is written in HTML.


Syntax

The Doctype element uses the tag <!DOCTYPE html>. This is the first tag in the document, and it does not have an end tag.

The HTML element uses the <html> tag. It has an end tag, as do all of the following elements (unless otherwise stated).


Example Code

<!DOCTYPE html>
  <html>
    (HTML code for website)
  </html>


Example Rendering

These two elements are not rendered, but they allow the rest of the website to be rendered.




Header and Title


Description

The header element includes the part of the document that provides meta information about how the document should be read and managed.

Inside the head element is the title element, which establishes the title of the website to be used by the browser.


Syntax

The header element uses the <head> tag, and the title element uses the <Title> tag.


Example Code

<head>
  <title>HTML Reference</title>
  <link href="index.css" rel="stylesheet" type="text/css" />
</head>


Example Rendering

These elements are not rendered.




Body, Text, and Spacing


Description

The body element contains the remainder of the HTML page, the part that will be rendered.

Within the body, several different elements can be used to thematically separate the page's textual content. The heading element indicates headings of various sizes and importance, while the paragraph element is used for the bulk of the text in a website.

Some elements take up a new line by default, but other line breaks written in the code will not be rendered, so a line break element must be used to indicate a desired line break.

The horizontal rule element creates a horizontal line that divides the page.


Syntax

The body, heading, paragraph, line break, and horizontal rule elements use the <Body>, <h1>, <p>, <br>, and <hr> tags, respetively.

The heading tag can have numbers from 1 to 6, with smaller numbers indicating more important headings. The line break and horizontal rule elements are empty elements; they do not have end tags.


Example Code

<body>
    <h4>Apples</h4>
    <p>An apple is a fruit. Apples can be red, yellow, or green.</p>
    <p>Apples can be used to make apple pie.</p><br>
    <h4>Bananas</h4>
    <p>Bananas are also fruits. Unripe bananas are green. Ripe bananas are yellow.</p>
    <p>Bananas can be used to make banana bread.</p><br>
    <hr>
    <h4>Carrots</h4>
    <p>Carrots are vegetables. They are orange.</p>
    <p>Carrots can be used to make carrot juice.</p>
</body>


Example Rendering

Apples

An apple is a fruit. Apples can be red, yellow, or green.

Apples can be used to make apple pie.


Bananas

Bananas are also fruits. Unripe bananas are green. Ripe bananas are yellow.

Bananas can be used to make banana bread.


Carrots

Carrots are vegetables. They are orange.

Carrots can be used to make carrot juice.

Links


Description

The link element allows for a part of one's website to direct a user to a different page. Any rendered HTML element, including text and images, can be a link. Links can direct users to external websites or to other pages within the same website.


Syntax

<a href="Link URL"> Content that will direct a user to the link when clicked </a>


Example Code

<a href="http://google.com">Google</a>


Example Rendering

Google




Images


Description

The image element is used to insert an image into a website. It is commonly used with several attributes to specify the characteristics of the image.


Syntax

Note: this is an empty element and has no end tag.

<img src="Image URL" alt="Alternate text if image fails to load"
height="Image height" width="Image width">


Example Code

<img src="https://images.landofnod.com/is/image/LandOfNod/Chair_Play_Parker_CE_SQ"
alt="Chair" height="80" width="80">


Example Rendering




Lists


Description

The list element can be used to display a series of related textual items in the form of bulleted, numbered, or lettered lists.


Syntax

Note: for an ordered list use "ol" instead of "ul."


<ul>
   <li>A listed item</li>
   <li>Another listed item</li>
   <li>Another listed item</li>
   <li>Another listed item</li>
</ul>


Example Code

<ul>
   <li>Apples</li>
   <li>Bananas</li>
   <li>Carrots</li>
   <li>Chairs</li>
</ul>


Example Rendering

  • Apples
  • Bananas
  • Carrots
  • Chairs



Tables


Description

The table element is used to display tables, which can be styled in many different ways.


Syntax

The entire table element is surrounded by the <table> tags. The elements in the table are written in HTML by row, with the <tr> tags surrounding the content of each row. The individual data points inside the rows are surrounded by <td> tags, or by <th> tags if they are table headers.


Example Code

<table width="200">
   <tr>
      <th>First Name</th>
      <th>Last Name</th>
   </tr>
      <td>Repulta</td>
      <td>Gobulmi</td>
   <tr>
      <td>Berlickens</td>
      <td>Schmaloop</td>
   </tr>
</table>


Example Rendering

First Name Last Name
Repulta Gobulmi
Berlickens Schmaloop



Blocks


Description

The block element can be used to group together several elements so that an attribute may be written only once but apply to the whole group of elements.


Syntax

The block element uses the <div> tag, which surrounds the items that make up the block.


Example Code

Note that the border attribute is only written in one place.

<div style="border:2px dotted black">
   <img src="https://vignette.wikia.nocookie.net/vsbattles/images/a/ac/Tree.png/revision/latest?cb=20170515233039" alt="tree" height="80" width="200">
   <p>   This is a tree.</p>
</div>


Example Rendering

   This is a tree.
